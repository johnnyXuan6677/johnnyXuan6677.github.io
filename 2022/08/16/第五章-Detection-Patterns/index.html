Patterns for Fault Tolerant software 第五章 Detection Patterns

The Overview of Detection Patterns

  • 兩種方法來偵測錯誤
    • priori detection - 透過利用在正常情況下系統的運作狀態(states),結果(result),副作用(side effect)觀察現行運作的系統是否有脫離正常行為模式的趨勢，進而做到提前預防的功能，若沒有偵測到任何裝況異常此系統將會完全不做任何動作。
    • redundant hardware-複製功能程式一模一樣的硬體設備
      • https://link.springer.com/chapter/10.1007/978-1-4614-2113-9_4
    • 另一種偵測方法，讓系統自己學習正確的運作模式並且讓系統擁有自我修復的能力(Bayesian learning techniques)
      • https://brohrer.mcknote.com/zhHant/statistics/how_bayesian_inference_works.html
      • 此系統需要同時擁有detect both errors and failures的能力尤其是error要比failure還重要得多，這是因為發生errors時這些的錯誤並還沒嚴重到會讓observers注意到
  • 在failure handling in computing system當中我們最想做到的是error可以自動被偵測並解被recover before it becomes a failure ,but if it is not possible when the failure occurs then we will prefer to use ‘fail-silent’ and the the crash failure mode (describe in CH1)
  • Detect error in two mechanisms
    • test function return codes to determine if the function has returned reoporting and error condition
    • use exception or try/catch function ,the mechanisms to detect error conditions exists.

    Fault Correlation

    What fault is activating ?

  • Question
    • what has the error done?
    • Has execution stopped?
    • What capabilities are no longer available?
    • what was the size of the stack at the time of the error?
    • Were logs collected?
    • What data is incorrect?
    • Is it frequerntly changing data or is it a constant?
  • 定義錯誤的痛點是一個有趣的問題，過於巨大的復原機制(recovery)會對系統造成巨大的影響，所以定義error的核心是很重要的。

Error Containment Barrier

How to avoid the spread of error !

  • Units of Mitigation

    What is the first thing that the system must do when it detects an error?

Heatbeats

Send a status report at regular intervals to let other parts of the system know that the sender is alive.

Acknowledgement

Send a reply message to let a communicating party know that the sender is alive.

WatchDog

Build a special entity to watch over another to make sur that it is still operting well
