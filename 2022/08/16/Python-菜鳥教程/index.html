Python 機器學習

監督式學習

從給定的訓練數據集中學習出一個模式（函數 / learning model），當新的數據到來時，可以根據這個模式預測結果。監督學習的訓練集要求是包括輸入和輸出，也可以說是特徵和目標，訓練集中的目標是由人標註(Label)的，常見的監督學習算法包括回歸分析和統計分類。(SVM, ANN  Deep Learning)例如：我們可以給定很多人的照片記錄他們相應的種族，我們可以訓練一個模型來分類從未看過的任意照片。

分類模型

Nearest Neighbors 分類演算法

  • K nearest neighbor(KNN)
  • Pros
    1. 超級簡單執行，很直觀
    2. 比linear regression SVM 還快速因為它沒有做訓練
    3. 新資料可以快速分類
  • Cons
    1. 不適合多維度分類，維度太多會增加欲分類點相對它的距離
    2. 不適合資料量過大的model，資料量愈多所需要計算距離的效能就要越大
  • KNN屬於機器學習中的監督式學習(Supervised learning)，不過一般來說監督式學習是透過資料訓練(training)出一個model，但是在KNN其實並沒有做training的動作。KNN一般用來做資料的分類，如果你已經有一群分好類別的資料，後來加進去點就可以透過KNN的方式指定新增加資料的分類。
  • KNN 解說(淺顯易懂) https://medium.com/@NorthBei/machine-learning-knn%E5%88%86%E9%A1%9E%E6%BC%94%E7%AE%97%E6%B3%95-b3e9b5aea8df

非監督式學習

訓練資料沒有標準答案、不需要事先以人力輸入標籤，故機器在學習時並不知道其分類結果是否正確。訓練時僅須對機器提供輸入範例，它會自動從這些範例中找出潛在的規則，常見的非監督學習算法包括聚類(Clustering)。

Reinforcement Learning

通過觀察來學習做成如何的動作，每個動作都會對環境有所影響，學習對象根據觀察到的周圍環境的反饋來做出判斷。比如老師會根據我的開心程度來打分, 我開心時，可以得到高分, 我不開心時得到低分. 有了這些被打分的經驗, 我就能判斷為了拿到高分, 我應該選擇一張開心的臉, 避免選到傷心的臉. 這也是強化學習的核心思想. 可以看出在強化學習中, 一種行為的分數是十分重要的. 所以強化學習具有分數導向性. 我們換一個角度來思考.這種分數導向性好比我們在監督學習中的正確標籤.
